# 大文件高效编辑指南

## 问题说明

当编辑大文件（如 500+ 行的文章）时，如果使用全文覆写的方式，AI 需要：
1. 读取整个文件（消耗输入 token）
2. 生成整个文件（消耗输出 token）

这会导致 token 消耗巨大，成本高且效率低。

## 解决方案：使用精准编辑

你的项目已经支持**精准编辑**功能，只需要修改文件的一部分，而不是重写整个文件。

---

## ⚠️ 关键问题：如何让 AI 只读取需要的部分？

### 问题描述

当你使用 `@文件路径:行号范围` 语法时，AI 可能会：
- ❌ 直接读取整个文件（消耗大量 token）
- ❌ 使用 `/workspace/file` API 读取全文

### 解决方案：明确指导 AI 的行为

#### 方法 1：明确指定只读取特定行（推荐）

**✅ 正确做法：**
```
请修改文件 d:\sailvan\读写测试文件夹\doc\星际觉醒_第一章.md
只读取第 45-50 行来确认内容，然后修改第 45 行的"量子剑"为"光能剑"
```

**AI 会这样操作：**
```typescript
// 1. 只读取需要的行范围
read_file({
  target_file: "d:\\sailvan\\读写测试文件夹\\doc\\星际觉醒_第一章.md",
  offset: 44,  // 从第 45 行开始（offset 从 0 开始，所以是 44）
  limit: 6     // 读取 6 行（45-50）
})

// 2. 然后使用 search_replace 进行修改
search_replace({
  file_path: "d:\\sailvan\\读写测试文件夹\\doc\\星际觉醒_第一章.md",
  old_string: "...量子剑...",
  new_string: "...光能剑..."
})
```

#### 方法 2：直接提供内容片段（最省 token）

**✅ 最佳做法：**
```
请修改文件 d:\sailvan\读写测试文件夹\doc\星际觉醒_第一章.md
找到这段内容（不需要读取文件，直接搜索替换）：
```
林清澜深吸一口气，量子剑在手中凝聚成形——那是由纯粹的光子编织而成的武器，随着她的意志而变化。
```
把"量子剑"改成"光能剑"
```

**AI 会直接操作：**
```typescript
// 不需要读取文件，直接使用 search_replace
search_replace({
  file_path: "d:\\sailvan\\读写测试文件夹\\doc\\星际觉醒_第一章.md",
  old_string: "林清澜深吸一口气，量子剑在手中凝聚成形——那是由纯粹的光子编织而成的武器，随着她的意志而变化。",
  new_string: "林清澜深吸一口气，光能剑在手中凝聚成形——那是由纯粹的光子编织而成的武器，随着她的意志而变化。"
})
```

#### 方法 3：使用搜索功能定位（适合不确定位置）

**✅ 推荐做法：**
```
请修改文件 d:\sailvan\读写测试文件夹\doc\星际觉醒_第一章.md
先搜索"量子剑在手中凝聚成形"来定位位置，然后只读取该位置附近的行（比如前后各 5 行），最后进行修改
```

**AI 会这样操作：**
```typescript
// 1. 先搜索定位
grep({
  pattern: "量子剑在手中凝聚成形",
  path: "d:\\sailvan\\读写测试文件夹\\doc\\星际觉醒_第一章.md"
})

// 2. 根据搜索结果，只读取相关行（假设在第 45 行）
read_file({
  target_file: "d:\\sailvan\\读写测试文件夹\\doc\\星际觉醒_第一章.md",
  offset: 40,  // 读取前后各 5 行
  limit: 11
})

// 3. 进行修改
search_replace({...})
```

### ❌ 避免的做法

**不要这样做：**
```
❌ 请修改这个文件：@d:\sailvan\读写测试文件夹\doc\星际觉醒_第一章.md:1-501
把第 50 行的"量子剑"改成"光能剑"
```

**原因：**
- `@文件路径:1-501` 这种语法会让 AI 认为需要读取整个文件
- AI 可能会调用 `/workspace/file` API，读取全部 501 行
- 消耗大量 token

**应该改为：**
```
✅ 请修改文件 d:\sailvan\读写测试文件夹\doc\星际觉醒_第一章.md
只读取第 48-52 行来确认内容，然后修改第 50 行的"量子剑"为"光能剑"
```

或者：
```
✅ 请修改文件 d:\sailvan\读写测试文件夹\doc\星际觉醒_第一章.md
找到包含"量子剑在手中凝聚成形"的句子，把"量子剑"改成"光能剑"（不需要读取全文）
```

## 方法一：使用 `search_replace` 工具（推荐）

### 适用场景
- 修改特定段落、句子或词语
- 需要替换的内容在文件中唯一或可以通过上下文精确定位

### 使用方式

#### 1. 提供最小化上下文

**❌ 错误做法：**
```
请修改这个文件：@d:\sailvan\读写测试文件夹\doc\星际觉醒_第一章.md:1-501
把第 50 行的"量子剑"改成"光能剑"
```

**✅ 正确做法：**
```
请修改文件 d:\sailvan\读写测试文件夹\doc\星际觉醒_第一章.md
找到这段内容：
```
林清澜深吸一口气，量子剑在手中凝聚成形——那是由纯粹的光子编织而成的武器，随着她的意志而变化。
```
把"量子剑"改成"光能剑"
```

或者更精确：
```
请修改文件 d:\sailvan\读写测试文件夹\doc\星际觉醒_第一章.md
找到包含"量子剑在手中凝聚成形"的段落，把"量子剑"改成"光能剑"
```

#### 2. AI 会使用 `search_replace` 工具

AI 会自动使用 `search_replace` 工具，只需要：
- 提供足够的上下文来唯一匹配目标内容（使用 `before` 和 `after` 参数）
- 指定要替换的 `oldText` 和 `newText`

**示例：**
```typescript
// AI 会自动生成类似这样的操作
search_replace({
  file_path: "d:\\sailvan\\读写测试文件夹\\doc\\星际觉醒_第一章.md",
  old_string: "林清澜深吸一口气，量子剑在手中凝聚成形——那是由纯粹的光子编织而成的武器，随着她的意志而变化。",
  new_string: "林清澜深吸一口气，光能剑在手中凝聚成形——那是由纯粹的光子编织而成的武器，随着她的意志而变化。"
})
```

## 方法二：使用行号范围编辑

### 适用场景
- 修改连续的多行内容
- 知道确切的行号范围

### 使用方式

#### 1. 提供行号和修改内容

**示例：**
```
请修改文件 d:\sailvan\读写测试文件夹\doc\星际觉醒_第一章.md
修改第 45-47 行，将内容改为：
```
林清澜深吸一口气，光能剑在手中凝聚成形。
她的身影瞬间消失在原地。
```

#### 2. AI 会使用精准编辑 API

AI 会调用 `/workspace/edits` API，使用 `range` 类型编辑：

```json
{
  "path": "doc/星际觉醒_第一章.md",
  "edits": [
    {
      "type": "range",
      "startLine": 45,
      "endLine": 47,
      "newText": "林清澜深吸一口气，光能剑在手中凝聚成形。\n她的身影瞬间消失在原地。"
    }
  ]
}
```

## 方法三：使用锚点编辑（最精确）

### 适用场景
- 需要替换的内容在文件中出现多次
- 需要通过前后文来精确定位

### 使用方式

**示例：**
```
请修改文件 d:\sailvan\读写测试文件夹\doc\星际觉醒_第一章.md
找到这段内容：
```
"第一阶段，开始。"

她的身影瞬间消失在原地。

下一刻，十七道剑光同时斩向BOSS的要害。
```
把中间的"她的身影瞬间消失在原地。"改成"她的身影如闪电般移动。"
```

AI 会使用 `anchor` 类型编辑，自动添加前后文作为锚点：

```json
{
  "path": "doc/星际觉醒_第一章.md",
  "edits": [
    {
      "type": "anchor",
      "before": "\"第一阶段，开始。\"\n\n",
      "oldText": "她的身影瞬间消失在原地。",
      "after": "\n\n下一刻，十七道剑光同时斩向BOSS的要害。",
      "newText": "她的身影如闪电般移动。"
    }
  ]
}
```

## Token 消耗对比

假设文件有 500 行，约 20,000 tokens：

### 全文覆写方式
- **输入 tokens**: ~20,000（读取全文）
- **输出 tokens**: ~20,000（生成全文）
- **总计**: ~40,000 tokens

### 精准编辑方式（修改 1 行）
- **输入 tokens**: ~500（只读取相关上下文）
- **输出 tokens**: ~100（只生成修改指令）
- **总计**: ~600 tokens

**节省**: 约 **98.5%** 的 token！

## 📋 实用指南：如何避免 AI 读取全文

### 场景 1：你知道确切的行号

**❌ 错误：**
```
修改文件：@文件路径:1-501
修改第 50 行
```

**✅ 正确：**
```
修改文件：文件路径
只读取第 48-52 行，然后修改第 50 行的"量子剑"为"光能剑"
```

### 场景 2：你不知道行号，但知道内容

**❌ 错误：**
```
修改文件：@文件路径:1-501
找到"量子剑"这个词并修改
```

**✅ 正确：**
```
修改文件：文件路径
搜索"量子剑在手中凝聚成形"这个句子，找到后把"量子剑"改成"光能剑"（不需要读取全文，直接使用 search_replace）
```

### 场景 3：你需要查看文件结构

**❌ 错误：**
```
读取文件：@文件路径:1-501
查看文件内容
```

**✅ 正确：**
```
读取文件：文件路径
只读取前 50 行查看文件结构
```

或者：
```
读取文件：文件路径
只读取文件头部（前 30 行）和文件尾部（后 20 行）来了解文件结构
```

### 场景 4：你需要修改多个位置

**❌ 错误：**
```
修改文件：@文件路径:1-501
把所有"量子剑"改成"光能剑"
```

**✅ 正确：**
```
修改文件：文件路径
使用 search_replace 工具，把所有"量子剑"替换为"光能剑"（使用 replace_all=true，不需要读取全文）
```

### 场景 5：你需要添加内容

**❌ 错误：**
```
修改文件：@文件路径:1-501
在第 100 行后面添加新内容
```

**✅ 正确：**
```
修改文件：文件路径
只读取第 98-102 行来确认位置，然后在第 100 行后面添加新内容
```

或者：
```
修改文件：文件路径
找到"战斗，开始了。"这句话，在它后面添加新内容（不需要读取全文）
```

## 🎯 核心原则总结

### 给 AI 的指令应该包含：

1. **明确的行号范围**（如果知道）
   ```
   只读取第 X-Y 行
   ```

2. **明确的搜索关键词**（如果不知道行号）
   ```
   搜索"关键词"，找到后修改（不需要读取全文）
   ```

3. **明确的操作方式**
   ```
   使用 search_replace 工具，不需要读取文件
   ```

4. **明确的限制**
   ```
   不要读取整个文件，只读取需要的部分
   ```

### 避免的指令：

1. ❌ `@文件路径:1-501` - 会让 AI 读取全文
2. ❌ `读取文件内容` - 没有限制，AI 会读取全文
3. ❌ `查看文件` - 太模糊，AI 可能读取全文
4. ❌ `修改文件` + 粘贴全文 - 浪费 token

---

## 总结

1. **使用 `search_replace` 工具**：最省 token，适合大多数场景
2. **只提供必要的上下文**：不要粘贴整个文件
3. **提供足够的定位信息**：确保能唯一匹配到目标内容
4. **使用行号（如果知道）**：最直接的方式
5. **明确指导 AI 的行为**：告诉 AI 只读取需要的部分
6. **避免全文覆写**：除非确实需要重写整个文件

遵循这些原则，可以大幅减少 token 消耗，提高编辑效率！

## 💡 额外提示

### 如果 AI 仍然读取了全文怎么办？

如果发现 AI 读取了全文，可以在后续对话中提醒：

```
请注意：这个文件有 500+ 行，请只读取需要的部分，不要读取全文。
如果不知道具体位置，请先使用 grep 搜索定位，然后只读取相关行。
```

或者：

```
请使用 search_replace 工具直接修改，不需要先读取文件。
目标内容：[粘贴需要修改的内容片段]
```

这样可以引导 AI 使用更高效的方式。
